## 一、硬件基础

### 单片机 / 微控制器 (MCU)
**一句话**: 一块集成了 CPU、内存、外设接口的"微型电脑芯片"，专门用来控制硬件。
**详解**: 和你的电脑(CPU+内存+硬盘+各种卡分开)不同，单片机把这些都塞进一块指甲盖大小的芯片里。它没有操作系统(或只有很小的实时系统)，开机就跑你写的程序，专门做一件事，省电、便宜、可靠。
**本项目**: ESP32-S3 就是单片机，负责读按键、转屏幕、发蓝牙。

### ESP32-S3
**一句话**: 乐鑫(Espressif)公司出的一款带 WiFi 和蓝牙的双核单片机。
**详解**: ESP32 是一个芯片家族，S3 是其中一款高性能型号，特点是双核 Xtensa LX7 CPU (主频 240MHz)，支持 WiFi 和蓝牙 5，还有 USB OTG 接口。它便宜、功能全，是 DIY 物联网项目的热门选择。
**本项目**: 整个键盘的大脑。选它是因为自带蓝牙(发快捷键)和 WiFi(做配置网页)，一块芯片全搞定。

### Xtensa
**一句话**: 一种由 Cadence 公司设计的高性能嵌入式 CPU 架构。
**详解**: CPU 能听懂的"命令语言"叫指令集。常见的有 ARM(手机)、x86(电脑)。Xtensa 是可配置的嵌入式架构，乐鑫 ESP32/ESP32-S3 系列都用它。ESP32-S3 的 CPU 就是 Xtensa LX7 架构。
**本项目**: 决定了我们用的编译器是 `xtensa-esp32s3-elf-gcc`(给 Xtensa 芯片编译程序的工具)。

### Flash (闪存)
**一句话**: 单片机的"硬盘"，断电后数据不丢，存放程序代码。
**详解**: 程序烧录进去就存在 Flash 里，断电也不会丢。ESP32-S3-N16R8 的 16MB 就是指 16MB Flash。
**本项目**: 存放固件程序。我们的程序占了约 1.4MB，Flash 用了 46.9%，还很宽裕。也用一小块存配置(NVS)。

### RAM (内存)
**一句话**: 单片机的"工作台"，运行时临时存数据，断电就清空。
**详解**: 程序运行时的变量、缓冲区都放在 RAM。ESP32-S3 有 512KB SRAM + 8MB PSRAM。RAM 比 Flash 小得多，要省着用。
**本项目**: 运行时占用约 50KB(15.3%)，BLE 和 WiFi 都比较吃 RAM。PSRAM 提供额外的大内存空间。

### GPIO (通用输入输出引脚)
**一句话**: 单片机上可编程控制的"小电线接口"，可以输出高/低电平或读取外部电平。
**详解**: GPIO = General Purpose Input/Output。每个引脚(pin)都有编号(如 GPIO7)。设为输出时可以控制 LED 亮灭、给屏幕发信号；设为输入时可以读按键按没按。高电平≈3.3V(逻辑1)，低电平≈0V(逻辑0)。
**本项目**: 用 GPIO 连屏幕(SPI)、读按键、读旋钮。`pins.h` 里就是引脚分配表。

### 上拉电阻 / 内部上拉 (Pull-up)
**一句话**: 让一个悬空的引脚默认保持高电平的小电路，避免读数乱跳。
**详解**: 按键没按下时，引脚是"悬空"的，读到的值会乱飘。加一个上拉电阻让它默认是高电平(1)，按下时接地变成低电平(0)，这样就能稳定判断。ESP32 内部自带上拉，代码里 `INPUT_PULLUP` 一行就启用，不用外接电阻。
**本项目**: 5 个按键和编码器都用内部上拉，按下=低电平。

### 电平 (高/低电平)
**一句话**: 用电压高低表示数字信号的 1 和 0。
**详解**: 3.3V 系统里，约 3.3V 算高电平(1)，约 0V 算低电平(0)。数字电路就靠这两个状态传递信息。
**本项目**: 按键按下把引脚从高(3.3V)拉到低(0V)，程序检测这个变化。

## 二、通信接口

### SPI (串行外设接口)
**一句话**: 一种高速的"芯片对芯片"通信方式，用几根线传数据。
**详解**: SPI = Serial Peripheral Interface。主要 4 根线:
- **MOSI**: 主→从 数据线 (Master Out Slave In)
- **MISO**: 从→主 数据线 (本项目屏幕只接收不用它)
- **SCLK/CLK**: 时钟线，控制数据节奏
- **CS**: 片选线，选中哪个设备 (Chip Select)

SPI 快(可达几十 MHz)，适合屏幕这种数据量大的设备。
**本项目**: 4 个屏幕都用 SPI。它们**共享** MOSI/CLK，但每个有**独立的 CS**——想跟哪个屏幕说话就把那个 CS 拉低。

### CS (片选 / Chip Select)
**一句话**: 多个设备共用一条总线时，用来"点名"当前跟谁通信的信号线。
**详解**: SPI 总线上可以挂多个设备，但同一时刻只能跟一个说话。每个设备有自己的 CS 线，拉低哪个就是选中哪个，其他设备会"装聋"。
**本项目**: 4 个屏幕共享数据线，靠 4 根独立 CS(GPIO8-11)轮流选中刷新，互不干扰。

### I2C
**一句话**: 另一种芯片间通信方式，只用 2 根线，但比 SPI 慢。
**详解**: I2C 用 SCL(时钟)+SDA(数据)两根线，靠"地址"区分设备，省线但速度低。本项目没用到，了解即可——它和 SPI 是嵌入式里最常见的两种接口。

### UART / 串口
**一句话**: 最经典的"两根线收发数据"通信，常用于调试打印和烧录。
**详解**: UART 用 TX(发)+RX(收)两根线。开发板的 USB 口内部就转成 UART，连到电脑后可以看程序打印的日志(`Serial.println`)，也用来烧录程序。
**本项目**: 调试全靠它——`pio device monitor` 看到的日志就是通过 UART/USB 传来的。

### 总线 (Bus)
**一句话**: 多个设备共用的一组数据线。
**详解**: 就像一条公交线路，多个站点(设备)共用。SPI 总线、I2C 总线都是这个意思。共用能省引脚，但要靠 CS 或地址来区分目标。
**本项目**: 4 屏共享一条 SPI 总线。

## 三、无线通信

### 蓝牙 (Bluetooth) 与 BLE
**一句话**: 短距离无线通信；BLE 是其中的"低功耗"版本。
**详解**: BLE = Bluetooth Low Energy(低功耗蓝牙)。和传统蓝牙(传音频那种)不同，BLE 省电、适合小数据量、按需连接的设备，比如鼠标、键盘、手环。
**本项目**: 键盘通过 BLE 连电脑，把快捷键"无线"发过去，不用插线。

### HID (人机接口设备)
**一句话**: 键盘、鼠标这类"输入设备"的统一标准。
**详解**: HID = Human Interface Device。电脑天生认识 HID 设备，插上键盘鼠标不用装驱动就能用。我们让 ESP32 "假装"成一个 HID 键盘，电脑就以为接了个真键盘。
**本项目**: 核心机制——ESP32 通过 BLE HID 让电脑以为它是无线键盘，于是发出去的快捷键能直接触发 Kiro 的功能。

### NimBLE
**一句话**: 一个轻量、省内存的蓝牙协议软件库。
**详解**: 实现蓝牙功能需要一大套软件(协议栈)。乐鑫芯片有两套: Bluedroid(功能全但占内存大)和 NimBLE(精简省内存)。ESP32-S3 推荐用 NimBLE。
**本项目**: 我们的 BLE 键盘库(HijelHID)就是基于 NimBLE 的。

### WiFi
**一句话**: 大家熟悉的无线上网技术，这里用来做配置网页。
**详解**: ESP32-S3 支持 2.4GHz WiFi。WiFi 有两种角色: 当"热点"(AP)让别人连，或当"客户端"(STA)去连别人的路由器。
**本项目**: 键盘开机变成一个 WiFi 热点，你手机连上去就能打开配置网页改键位。

### AP 模式 / STA 模式
**一句话**: AP = 自己当热点让别人连；STA = 自己去连别人的热点。
**详解**:
- **AP (Access Point)**: 设备自己发出一个 WiFi 信号，别的设备来连它。
- **STA (Station)**: 设备作为客户端去连接现成的路由器 WiFi。
**本项目**: 当前用 AP 模式(键盘自己当热点 "KiroKeyboard-Config")，好处是没有路由器也能配置。未来可选 STA 模式接入家里 WiFi。

### mDNS
**一句话**: 让你用好记的名字(如 kirokb.local)代替 IP 地址访问设备。
**详解**: mDNS = Multicast DNS。正常访问设备要记 IP(如 192.168.4.1)，mDNS 能让你直接用 `kirokb.local` 这种域名访问，更友好。
**本项目**: 规划中的 STA 模式会用到，当前 AP 模式直接用 192.168.4.1。

## 四、显示屏相关

### LCD
**一句话**: 液晶显示屏，本项目按键上那个小彩屏。
**详解**: LCD = Liquid Crystal Display。我们用的是 0.85 寸、128×128 像素的小彩屏，每个按键上一块，用来显示这个键的功能。

### IPS
**一句话**: 一种高质量的液晶屏技术，可视角度大、颜色好。
**详解**: IPS 是 LCD 的一种面板类型，特点是斜着看也不偏色。我们的 ScreenKey 模块就是 IPS 屏。
**本项目**: 代码里 ST7735 的 `ips=true` 参数就是告诉驱动这是 IPS 屏。

### ST7735
**一句话**: 驱动这块小屏幕的"显示控制芯片"型号。
**详解**: 屏幕本身不会自己显示，需要一个驱动芯片接收单片机的指令、控制每个像素。ST7735 就是 ScreenKey 模块里的这颗驱动芯片。不同屏幕用不同驱动芯片(如 ST7789、GC9107)，代码要选对型号。
**本项目**: 我们用 `Arduino_ST7735` 这个驱动类。选错型号屏幕就花屏或不亮。

### 像素 / 分辨率
**一句话**: 屏幕由很多小点(像素)组成，分辨率就是横竖各多少个点。
**详解**: 128×128 表示横 128 点、竖 128 点，共 16384 个像素。每个像素能独立显示颜色，点越多越清晰。
**本项目**: 每块屏 128×128，足够显示一个图标或几个字。

### RGB565 / 色深
**一句话**: 一种用 16 位(2字节)表示一个颜色的编码方式。
**详解**: 颜色由红(R)绿(G)蓝(B)三原色混成。RGB565 表示红用 5 位、绿用 6 位、蓝用 5 位，共 16 位，能表示 65536 种颜色(俗称 65K 色)。绿色多 1 位是因为人眼对绿色更敏感。
**本项目**: `display.h` 里 `COLOR_RED = 0xF800` 这些就是 RGB565 颜色值。

### 背光 (Backlight / BL)
**一句话**: 屏幕背后的灯，不开背光屏幕是黑的。
**详解**: LCD 自己不发光，靠背后一排 LED 照亮才能看见。BL 引脚控制背光开关(甚至亮度)。
**本项目**: BL 接 GPIO3，开机拉高点亮。如果屏幕全黑，先查背光。

### LVGL (备注)
**一句话**: 一个做漂亮图形界面的库(本项目暂未使用)。
**详解**: LVGL 能画按钮、图标、动画等精美 UI，但占资源多、配置复杂。本项目当前用更轻量的 Arduino_GFX 直接画文字色块，够用。未来想做花哨界面可换 LVGL。

## 五、输入设备

### 旋转编码器 (Rotary Encoder)
**一句话**: 可以无限旋转的旋钮，能检测转动方向和格数。
**详解**: 和音量旋钮(电位器)不同，旋转编码器没有终点，可以一直转。它内部通过 A、B 两相信号的先后顺序判断你是顺时针还是逆时针转，转一格发出几个脉冲。
**本项目**: 用 EC11 编码器做滚动/撤销/切换标签等连续操作。

### EC11
**一句话**: 最常见的一款旋转编码器型号，带按压功能。
**详解**: EC11 是个具体型号，5 个引脚: A/B 两相 + 公共地，外加一个按钮(按下旋钮)。
**本项目**: 旋转→发快捷键；短按→切换模式；长按→保存。

### 正交 / 象限 (Quadrature)
**一句话**: 编码器用两路错开的信号来判断旋转方向的原理。
**详解**: A、B 两相信号像两条错开的波。顺时针转时 A 先变，逆时针时 B 先变，靠这个先后差异判断方向。"正交"就是指这两相相差 90 度。
**本项目**: 不用自己算，库帮我们处理，只要接对 A/B 线。

### PCNT (脉冲计数器)
**一句话**: ESP32 芯片内部专门数脉冲的硬件模块。
**详解**: PCNT = Pulse Counter。编码器转动会产生脉冲，靠 CPU 一个个数会很累还可能漏。PCNT 是芯片里的专用硬件，在后台自动数，CPU 只要随时问"现在数到几了"。又快又准不丢步。
**本项目**: RotaryEncoderPCNT 库用 PCNT 硬件计数旋钮，比软件中断方式可靠。

### 消抖 (Debounce)
**一句话**: 过滤机械按键按下瞬间的电平抖动，避免一次按键被识别成多次。
**详解**: 机械触点闭合瞬间会"啪啪啪"弹跳几下(几毫秒)，电平快速抖动。如果不处理，一次按键可能被读成好几次。消抖就是"电平稳定超过 50ms 才算数"。
**本项目**: `buttons.cpp` 里用时间戳消抖，阈值 `KEY_DEBOUNCE_MS=50`。

### 短按 / 长按
**一句话**: 按一下就放=短按；按住超过设定时间=长按，可以触发不同功能。
**详解**: 同一个按键，按一下和按住能干不同的事。靠记录按下时长来区分(本项目 800ms 为界)。
**本项目**: 旋钮短按切模式、长按保存文件。

## 六、软件开发工具

### Arduino (框架)
**一句话**: 一套让单片机编程变简单的"傻瓜化"开发框架。
**详解**: 原本给单片机写程序很底层、很复杂。Arduino 把常用操作封装成简单函数(如 `digitalWrite()` 控制引脚、`Serial.println()` 打印)，还有海量现成库，新手也能快速上手。它不是某块板子，而是一套编程方式。
**本项目**: 我们用 Arduino 框架开发，因为快、库多、适合做原型。

### ESP-IDF
**一句话**: 乐鑫官方的专业开发框架，更底层、更强大但更难。
**详解**: IDF = IoT Development Framework。它比 Arduino 更接近硬件、控制更精细、适合量产产品，但学习曲线陡、开发慢。
**本项目**: 没用它(用了 Arduino)。但 Arduino 在 ESP32 上其实是建在 ESP-IDF 之上的。

### PlatformIO
**一句话**: 一个管理嵌入式项目的工具，自动搞定编译器、库、烧录。
**详解**: 写嵌入式程序需要编译器、各种库、烧录工具等一大堆东西。PlatformIO 是个"项目管家"，在 VS Code/Kiro 里用，自动下载安装这些工具，一条命令就能编译烧录。配置写在 `platformio.ini` 里。
**本项目**: 我们的构建工具。`pio run` 编译、`pio run -t upload` 烧录。

### pioarduino
**一句话**: PlatformIO 官方对 ESP32 + Arduino 3.x 组合支持不完善，社区做的替代版本。
**详解**: 官方 PlatformIO 的 ESP32 平台更新慢，不完全支持 ESP32-S3+最新 Arduino Core 3.x。社区 fork 了一个叫 pioarduino 的版本补上支持。
**本项目**: `platformio.ini` 里 `platform = https://github.com/pioarduino/...` 就是用它。

### 编译 (Compile)
**一句话**: 把人写的代码(C++)翻译成芯片能执行的机器码的过程。
**详解**: 你写的是给人读的 C++ 代码，芯片只懂 0/1 机器码。编译器负责翻译。翻译过程会检查语法错误。
**本项目**: `pio run` 就是编译。第一次编译要翻译几千个文件(包括库)，所以慢(几分钟)。

### 工具链 (Toolchain) / 编译器
**一句话**: 针对特定芯片的一整套翻译工具(编译器+链接器等)。
**详解**: 不同 CPU 架构需要不同的编译器。Xtensa 芯片要用 Xtensa 工具链(`xtensa-esp32s3-elf-gcc`)。工具链 = 编译器 + 汇编器 + 链接器 + 标准库等一套工具。
**本项目**: 用 `toolchain-xtensa-esp32s3`。环境搭建时它的目录结构出过问题(见 dev-environment 笔记)。

### 链接 (Link)
**一句话**: 把编译出的多个零件拼成一个完整可执行程序的过程。
**详解**: 每个源文件单独编译成一个"零件"(.o)，链接器把所有零件和库拼成最终的固件(.elf/.bin)。
**本项目**: 编译日志最后的 `Linking firmware.elf` 就是这步。

### 固件 (Firmware)
**一句话**: 烧录进单片机 Flash 里运行的那个程序文件。
**详解**: 编译链接的最终产物(firmware.bin)就是固件。把它"烧录"到芯片 Flash 里，芯片上电就运行它。
**本项目**: `firmware.bin` 是主程序，还有 `bootloader.bin`(开机引导)、`partitions.bin`(分区表)一起烧。

### 烧录 (Flash / Upload)
**一句话**: 把固件写进单片机的过程。
**详解**: 通过 USB 把编译好的固件传进芯片 Flash。烧完芯片重启就跑新程序。
**本项目**: `pio run -t upload`。底层用 esptool 工具。

### esptool
**一句话**: 乐鑫芯片专用的烧录/打包工具。
**详解**: 负责把固件写进 ESP32 的 Flash，也负责把 elf 转成 bin、合并多个 bin。
**本项目**: 环境里 esptool 出过版本问题(见 dev-environment)，最终用 5.3.0。

### 分区表 (Partition Table)
**一句话**: 规划 Flash 这块"硬盘"怎么分区使用的表格。
**详解**: Flash 要分成几块: 放程序的、放配置(NVS)的、放文件(LittleFS)的等。分区表定义每块多大、放哪。
**本项目**: `partitions/default_16MB.csv`，16MB 分给程序(双份支持OTA)、NVS、文件系统。

## 七、程序与库

### 库 (Library)
**一句话**: 别人写好的、可直接拿来用的代码包。
**详解**: 不用什么都从零写。比如驱动屏幕、发蓝牙这些复杂功能，有人写好了打包成库，你引入就能用。
**本项目**: 用了 5 个主要库——屏幕(Arduino_GFX)、蓝牙(NimBLE+HijelHID)、编码器(RotaryEncoderPCNT)、网页(ESPAsyncWebServer)、JSON(ArduinoJson)。

### 库依赖 (Dependency)
**一句话**: 一个库运行需要另一个库，这种"依赖"关系。
**详解**: 比如 HijelHID 键盘库依赖 NimBLE 库。PlatformIO 会自动帮你把依赖一起装上。
**本项目**: `platformio.ini` 的 `lib_deps` 列出依赖，工具自动下载。

### 兼容性 (Compatibility)
**一句话**: 某个库能不能在你的芯片/框架版本上正常工作。
**详解**: 库是别人写的，不一定支持所有芯片。ESP32-S3 + Arduino Core 3.x 组合下，很多老库没适配，编译会报错。要找支持新版 Core 的版本或替代库。
**本项目**: 踩过坑——TFT_eSPI、老版蓝牙库、老版编码器库都不兼容 C6，换成了兼容的替代品(见 dev-environment 笔记)。

### 头文件 (.h) 与源文件 (.cpp)
**一句话**: .h 是"目录/说明书"(声明有什么)，.cpp 是"正文"(具体怎么实现)。
**详解**: C++ 习惯把"接口声明"放头文件(.h)，"实现细节"放源文件(.cpp)。别的代码 `#include` 头文件就知道能调用哪些功能。
**本项目**: 每个模块都是一对，如 `buttons.h`(声明) + `buttons.cpp`(实现)。

### 模块化 (Modular)
**一句话**: 把程序拆成职责单一的小块，各管一摊，方便维护。
**详解**: 不把所有代码堆在一个文件，而是按功能拆: 按键归按键、屏幕归屏幕、蓝牙归蓝牙。改一处不影响其他，也好排查。
**本项目**: src 下分了 keymap/buttons/encoder/ble_hid/display/webconfig 等模块。

### 固件大小 / 资源占用
**一句话**: 程序占了多少 Flash(空间)和 RAM(内存)。
**详解**: 单片机资源有限，要关注程序占用。编译完会报告占比。
**本项目**: Flash 46.9%、RAM 15.3%，余量充足。

## 八、数据存储

### NVS (非易失性存储)
**一句话**: Flash 里专门存"小配置"的一块区域，断电不丢。
**详解**: NVS = Non-Volatile Storage。用"键值对"方式存少量数据(如设置、计数器)，断电后还在。比直接操作 Flash 简单安全。
**本项目**: 用户在网页改的按键映射存进 NVS，重启后保留。代码里 `Preferences` 库操作 NVS。

### LittleFS
**一句话**: 在 Flash 上模拟出的一个小型"文件系统"，能存文件。
**详解**: 让你像在电脑上一样在 Flash 里存文件(如网页 HTML、图片)。比 NVS 适合存大点的文件。
**本项目**: 分区里预留了 LittleFS。当前网页是直接嵌在程序里(PROGMEM)，没用 LittleFS，未来网页变大可改用它。

### PROGMEM
**一句话**: 把数据(如网页 HTML)直接存在 Flash 里，不占宝贵的 RAM。
**详解**: 大段固定数据(如一整个 HTML 页面)如果放 RAM 会浪费内存。PROGMEM 让它留在 Flash，用的时候再读。
**本项目**: `webpage.h` 的配置页 HTML 用 PROGMEM 存，省 RAM。

### JSON
**一句话**: 一种人和机器都能读的轻量数据格式，用来交换结构化数据。
**详解**: JSON 长这样: `{"name":"Key1","key":"L"}`。它清晰、通用，网页和设备之间传数据基本都用它。
**本项目**: 按键映射用 JSON 表示，网页和设备之间、存 NVS 都用 JSON。用 ArduinoJson 库解析/生成。

### 序列化 / 反序列化 (Serialize / Deserialize)
**一句话**: 把内存里的数据变成可保存/传输的文本(序列化)，再变回来(反序列化)。
**详解**: 内存里的数据结构没法直接存文件或网络传输，要先"打包"成 JSON 文本(序列化)。收到后再"解包"成数据结构(反序列化)。
**本项目**: `keymapToJson()` 序列化、`keymapFromJson()` 反序列化。

## 九、Web 与网络

### Web 服务器 (Web Server)
**一句话**: 一个能响应浏览器请求、返回网页的程序。
**详解**: 你浏览器输入网址，背后有个服务器收到请求、返回页面。ESP32 虽小，也能跑一个迷你 Web 服务器。
**本项目**: ESP32 上跑 Web 服务器，你手机浏览器访问它就能打开配置页。

### 异步 (Async)
**一句话**: 不傻等、能同时处理多件事的工作方式。
**详解**: "同步"是排队办事(办完一个才下一个)。"异步"是能在等待时去干别的。异步 Web 服务器能一边处理网页请求一边不耽误键盘扫描。
**本项目**: 用 ESPAsyncWebServer(异步版)，不会因为网页请求卡住按键响应。

### HTTP
**一句话**: 浏览器和服务器之间通信的"语言规则"。
**详解**: 你访问网页时，浏览器用 HTTP 协议向服务器发请求(GET 取数据、POST 提交数据)，服务器用 HTTP 回应。
**本项目**: 配置页通过 HTTP 和设备通信。

### REST API
**一句话**: 一套用网址+方法来读写数据的接口规范。
**详解**: API 是程序间的接口。REST 风格用网址表示资源、用 HTTP 方法表示操作:
- `GET /api/config` = 读取配置
- `POST /api/config` = 提交新配置
**本项目**: 网页通过这些 API 和设备交换按键映射。

### GET / POST
**一句话**: HTTP 的两种常用请求——GET 取数据，POST 发数据。
**详解**: GET 像"查询"(给我看看配置)，POST 像"提交"(把新配置存进去)。
**本项目**: 网页加载时 GET 配置，点保存时 POST 新配置。

### 前端 / 后端
**一句话**: 前端=用户看到的网页界面；后端=设备上处理数据的程序。
**详解**: 前端(HTML/CSS/JS)跑在你浏览器里，负责显示和交互。后端跑在 ESP32 上，负责存取数据。两者通过 API 沟通。
**本项目**: 前端是 `webpage.h` 的 HTML；后端是 `webconfig.cpp`。

### IP 地址
**一句话**: 网络中每个设备的"门牌号"。
**详解**: 连上网络的设备都有个 IP(如 192.168.4.1)，靠它互相找到对方。
**本项目**: 键盘做热点时 IP 是 192.168.4.1，浏览器访问这个地址打开配置页。

## 十、编程概念

### 变量 / 常量
**一句话**: 变量是能改的存储格子，常量是固定不变的值。
**详解**: 变量存运行时会变的数据(如当前模式)；常量是写死的设定(如引脚号)。本项目常量多用 `#define` 或 `const` 定义在 config.h/pins.h。

### 函数 (Function)
**一句话**: 一段有名字、可重复调用的代码块。
**详解**: 把一件事的步骤打包成函数，需要时调用，避免重复写。如 `bleHidSendAction()` 封装了发送按键的逻辑。

### 数组 (Array)
**一句话**: 一排同类型数据的集合，用编号访问。
**详解**: 比如 4 个按键的动作存在 `g_keyActions[4]` 数组里，`g_keyActions[0]` 是第一个键。

### 结构体 (struct)
**一句话**: 把相关的几个数据打包成一个自定义类型。
**详解**: 比如一个按键动作有"类型、修饰键、键码、标签"几项，用 `struct KeyAction` 打包成一个整体。

### 枚举 (enum)
**一句话**: 给一组固定取值起好记的名字。
**详解**: 比如动作类型只有几种，用枚举 `ActionType::HOTKEY` 比用数字 1 更易读。

### 指针 (Pointer)
**一句话**: 存"数据在哪"的地址，而不是数据本身。
**详解**: 嵌入式常用指针节省内存。`const char* label` 存的是文字的地址。新手只需知道它指向某块数据即可。

### 回调函数 (Callback)
**一句话**: 提前登记一个函数，让别人在特定时刻替你调用。
**详解**: 比如告诉 Web 模块"配置变了就调用我这个函数",它在用户保存时自动帮你执行刷新屏幕。
**本项目**: `onConfigChanged` 就是回调，Web 保存后自动刷新屏幕显示。

### 主循环 (loop)
**一句话**: 单片机程序里不断重复执行的核心循环。
**详解**: Arduino 程序有两部分: `setup()` 开机跑一次(初始化)，`loop()` 之后无限重复。所有持续性工作(扫按键、读旋钮)都在 loop 里。
**本项目**: `main.cpp` 的 `loop()` 不停扫描按键和旋钮、发送动作。

### 中断 (Interrupt)
**一句话**: 硬件事件发生时打断主程序、立即处理的机制。
**详解**: 与其让 CPU 不停轮询"按键按了吗",不如让硬件在事件发生时主动"敲门"通知 CPU。响应更及时。
**本项目**: 编码器的 PCNT 在硬件层用类似机制后台计数，我们的按键则用轮询+消抖。

### 轮询 (Polling)
**一句话**: 主程序反复主动检查"状态变了吗"。
**详解**: 和中断相反，轮询是循环里一遍遍查。简单但可能浪费 CPU。对按键这种不频繁的事件够用。
**本项目**: 按键扫描用轮询(每 5ms 查一次)。
