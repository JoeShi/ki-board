阶段 1
产品调研
在小红书、淘宝、YouTube、GitHub 上搜索已有产品。了解 Stream Deck、DIY 宏键盘、ESP32 键盘项目的现状。发现市面上没有面向 AI agentic coding 的专用控制器。
你做 小红书 淘宝 market-research.md
阶段 2
Idea 探讨
和 Kiro 对话 3-4 轮。从模糊的痛点出发，Kiro 主动提问、挑战假设、帮你说"不做什么"，最终收敛为一份清晰的 Idea 文档。
你做 Kiro 引导 Idea.md
阶段 3
硬件选型
Kiro 帮你在淘宝搜索候选模块，对比规格/价格/兼容性，输出 BOM 采购清单。总成本约 300 元：ESP32-S3 + 3x ScreenKey + 圆形LCD + 矩形LCD。
Kiro 分析 淘宝 你下单 hardware-selection.md
阶段 4
接线设计
Kiro 根据选定硬件生成 GPIO 分配表、双 SPI 总线规划、面包板接线指南。如果有 KiCad MCP，还能直接生成原理图。
Kiro 生成 KiCad (可选) pins.h 硬件调试指南.md
阶段 5
环境搭建
Kiro 创建 platformio.ini、项目结构、库依赖。过程中遇到工具链解压嵌套、esptool 版本、库不兼容等问题，全部记录到 steering 文件。
Kiro 配置 PlatformIO platformio.ini dev-environment.md
阶段 6
硬件验证
硬件到货！按 A→G 七步逐项验证：上电 → 单屏 → 多屏 → 圆屏 → 矩形屏 → 按键 → USB HID。发现并解决了多屏 SPI 共享的 CS 片选关键问题。
你接线 Kiro 写测试 test/ 验证程序
阶段 7
固件开发 (Spec 驱动)
用 5 个 Spec 按依赖顺序推进：硬件驱动层 → UI 渲染层 → 状态机与按键 → Agent 注册 + Hook → BLE + 配对 + OTA。每个 Spec 由 Kiro 生成需求→设计→Tasks→代码。
Kiro Spec 你审核 src/ 全部固件代码
阶段 8
Companion App
用一个大 Spec 实现 Tauri 桌面应用：USB/BLE 双 transport、Kiro Hook 监听、Doubao ASR 语音识别、OTA 刷写、设置管理。Rust 后端 + React 前端。
Kiro Spec Tauri + Rust companion/ 目录
阶段 9
产品化
CI/CD 自动化：push to main 自动 bump 版本、编译固件+App、创建 GitHub Release。支持 OTA 空中升级，用户无需再手动烧录。
Kiro 生成 GitHub Actions release.yml
