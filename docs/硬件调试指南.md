## 二、接线表

### 2.1 引脚总览 (ESP32-C6 GPIO)

| 功能 | GPIO | 连接到 | 是否共享 |
|------|------|--------|---------|
| SPI MOSI (LCD_DIN) | GPIO7 | 4个屏幕的 DIN | 共享 |
| SPI CLK  (LCD_CLK) | GPIO6 | 4个屏幕的 CLK | 共享 |
| LCD_DC | GPIO4 | 4个屏幕的 DC | 共享 |
| LCD_RST | GPIO5 | 4个屏幕的 RST | 共享 |
| LCD_BL | GPIO3 | 4个屏幕的 BL (背光) | 共享 |
| LCD_CS0 | GPIO8 | 屏幕1 的 CS | 独立 |
| LCD_CS1 | GPIO9 | 屏幕2 的 CS | 独立 |
| LCD_CS2 | GPIO10 | 屏幕3 的 CS | 独立 |
| LCD_CS3 | GPIO11 | 屏幕4 的 CS | 独立 |
| KEY0 | GPIO0 | 屏幕1 的 KEY (按键) | 独立 |
| KEY1 | GPIO1 | 屏幕2 的 KEY | 独立 |
| KEY2 | GPIO2 | 屏幕3 的 KEY | 独立 |
| KEY3 | GPIO15 | 屏幕4 的 KEY | 独立 |
| EC11_A | GPIO18 | 编码器 A 相 | - |
| EC11_B | GPIO19 | 编码器 B 相 | - |
| EC11_SW | GPIO20 | 编码器按钮 | - |
| 3.3V | 3V3 | 所有模块 VCC | 电源 |
| GND | GND | 所有模块 GND + 编码器公共脚 | 地 |

> 引脚定义在 `include/pins.h`，若实际接线不同请同步修改。

### 2.2 ScreenKey Module 9PIN 定义 (参考 Waveshare)

每个屏幕模块的 GH1.25 9PIN: `VCC, GND, DIN, CLK, CS, DC, RST, BL, KEY`
- VCC → 3.3V
- GND → GND
- DIN → GPIO7 (共享)
- CLK → GPIO6 (共享)
- CS → 各自独立 (GPIO8/9/10/11)
- DC → GPIO4 (共享)
- RST → GPIO5 (共享)
- BL → GPIO3 (共享)
- KEY → 各自独立 (GPIO0/1/2/15)

### 2.3 EC11 编码器接线

EC11 通常 5 脚: 一侧 3 脚 (A, C/GND, B) 是编码器, 另一侧 2 脚是按钮。
- A 相 → GPIO18
- C (中间公共) → GND
- B 相 → GPIO19
- 按钮一脚 → GPIO20
- 按钮另一脚 → GND

> 代码使用内部上拉, 按下/旋转时拉低, 无需外接电阻。

### 2.4 接线注意事项

- **共享 SPI 线**: MOSI/CLK 需要并接到 4 个屏幕。面包板上可用一列做总线分配。
- **电源**: 4 个屏幕同时点亮的电流不小, 优先用开发板的 3.3V 输出。若供电不足导致花屏/重启, 改用独立 3.3V 电源, 并与 ESP32 共地。
- **线长**: SPI 27MHz 下杜邦线尽量短 (<15cm), 过长易花屏。花屏时可先降 SPI 频率 (见五-3)。

## 三、烧录步骤

### 3.1 编译并烧录

确保开发板通过 USB 连好。在项目目录运行:

```powershell
# 编译
pio run

# 编译 + 烧录 (自动识别串口)
pio run --target upload

# 指定串口烧录 (如 COM5)
pio run --target upload --upload-port COM5
```

> 若 `pio` 命令不识别, 用完整路径:
> `& "$env:USERPROFILE\.platformio\penv\Scripts\pio.exe" run --target upload`

### 3.2 查看串口日志

```powershell
pio device monitor --baud 115200
```

正常启动应看到:
```
====================================
  Kiro Keyboard booting...
====================================
[KEYMAP] No saved config, loading defaults
[MAIN] Keymap loaded
[BTN] Initialized 5 buttons
[ENC] Encoder initialized
[LCD] 4/4 screens initialized
[BLE] HID keyboard advertising as: Kiro KB
[WEB] AP: KiroKeyboard-Config  IP: 192.168.4.1
[WEB] Server started. Open http://192.168.4.1
[MAIN] Setup complete. Waiting for BLE connection...
```

## 四、分阶段验证 (建议顺序)

逐项验证, 出问题时容易定位。建议先单独接 1 个屏幕, 再扩展到 4 个。

### 阶段 A: 主控上电
- [ ] USB 连接后串口有启动日志
- [ ] 无反复重启 (重启通常是供电不足或接线短路)

### 阶段 B: 单屏显示
- [ ] 先只接 屏幕1 (CS0=GPIO8), 看是否点亮并显示 "AI Chat"
- [ ] 颜色正常 (不偏色)、无错位
- [ ] 若错位 → 调 ST7735 偏移 (见五-1)
- [ ] 若偏色/反色 → 调 IPS/BGR 设置 (见五-2)

### 阶段 C: 四屏显示
- [ ] 接全 4 个屏幕, 各显示对应标签 (AI Chat / Accept / Reject / Command)
- [ ] 4 屏内容不串、不闪 → 串扰说明 CS 切换或供电问题

### 阶段 D: 按键
- [ ] 按各屏幕按键, 串口打印 `[MAIN] KeyN short press`
- [ ] 屏幕有按下闪烁反馈
- [ ] 长按按键 (>800ms) 不误触发短按

### 阶段 E: 旋钮
- [ ] 旋转, 串口打印 `[MAIN] Encoder CW/CCW`
- [ ] 顺时针/逆时针方向正确 (反了就交换 A/B 接线, 或调代码)
- [ ] 一格一档, 不跳格/不丢档 → 调 COUNTS_PER_DETENT (见五-4)
- [ ] 短按切换模式 (串口打印模式名)
- [ ] 长按触发保存动作

### 阶段 F: BLE HID
- [ ] PC/手机蓝牙搜索到 "Kiro KB"
- [ ] 配对成功, 串口显示连接
- [ ] 打开文本编辑器, 按 Key2 (Accept=Tab) 看是否输入 Tab
- [ ] 在 Kiro/VSCode 中验证各快捷键实际效果

### 阶段 G: Web 配置
- [ ] 手机/PC WiFi 连接热点 "KiroKeyboard-Config"
- [ ] 浏览器打开 http://192.168.4.1 加载配置页
- [ ] 修改某键 (如改键码或标签), 点"保存并应用"
- [ ] 对应屏幕标签实时更新
- [ ] 重启设备, 配置依然保留 (NVS 生效)

## 五、参数微调点

### 1. ST7735 屏幕偏移 (显示错位时)
文件: `src/display.cpp`
```cpp
#define ST7735_COL_OFFSET  2   // 列偏移, 错位时试 0/1/2/3
#define ST7735_ROW_OFFSET  3   // 行偏移, 错位时试 0/1/2/3
```
128x128 的 ST7735 常见偏移组合: (2,3)、(2,1)、(0,0)。逐个试。

### 2. 颜色异常 (反色/偏色)
文件: `src/display.cpp` 的 Arduino_ST7735 构造参数
- 反色 (黑白颠倒): `ips` 参数改 `false`/`true` 试
- 红蓝颠倒: ST7735 构造最后的 `bgr` 参数 (当前默认 true), 可显式传 `false`
- 整体反色: 可调用 `invertDisplay(true/false)`

### 3. SPI 频率 (花屏时降速)
文件: `src/display.cpp`, `s_gfx[i]->begin()` 可传入速度:
```cpp
s_gfx[i]->begin(40000000);  // 默认约27-40MHz, 花屏时降到 20000000 或 10000000
```

### 4. 编码器每档计数 (跳格/丢档时)
文件: `src/encoder_input.cpp`
```cpp
#define COUNTS_PER_DETENT  4   // EC11 常见为4, 跳格改2, 丢档改1
```
旋一格如果跳 2 个事件 → 调大; 旋一格没反应 → 调小。

### 5. 按键消抖 / 长按阈值
文件: `include/config.h`
```cpp
#define KEY_DEBOUNCE_MS     50    // 抖动误触发时增大
#define KEY_LONG_PRESS_MS   800   // 长按判定时长
```

### 6. 按键映射默认值
文件: `src/keymap.cpp` 的 `keymapLoadDefaults()`
(也可上电后通过 Web 界面改, 存入 NVS)

## 六、常见故障排查

| 现象 | 可能原因 | 排查 |
|------|---------|------|
| 反复重启 | 供电不足 / 短路 | 减少同时点亮屏幕数; 检查 GND; 用独立3.3V |
| 屏幕全黑 | 背光/接线/CS错 | 确认 BL=GPIO3 拉高; 检查 DIN/CLK/CS; 单屏先测 |
| 屏幕花屏 | SPI过快/线太长 | 降 SPI 频率; 缩短杜邦线 |
| 显示错位 | ST7735偏移 | 调 COL/ROW_OFFSET |
| 颜色不对 | IPS/BGR设置 | 调 ips/bgr 参数或 invertDisplay |
| 4屏内容串扰 | CS切换/共享线 | 确认每屏CS独立; 检查总线接触 |
| 按键无反应 | 接线/GPIO | 万用表测按下是否拉低; 查 pins.h |
| 旋钮方向反 | A/B接反 | 交换 EC11_A/B 接线 |
| 旋钮跳格 | 每档计数 | 调 COUNTS_PER_DETENT |
| 搜不到蓝牙 | BLE未启动 | 看串口 [BLE] 日志; 重新配对前先在系统里删除旧配对 |
| 蓝牙连上但不输入 | 主机端 | 确认主机键盘布局为 US; 重新配对 |
| 连不上WiFi配置页 | AP/IP | 确认连的是 KiroKeyboard-Config; 地址 192.168.4.1 |

## 七、后续可选优化 (硬件验证通过后)

- **安全加固**: 给 WiFi AP 加密码 (`config.h` 的 WIFI_AP_PASS); 给 /api/config POST 加令牌校验
- **STA 模式**: 让设备加入家庭WiFi, 通过 mDNS (http://kirokb.local) 访问, 见 design-decisions
- **低功耗**: 利用 HijelHID 的 beforeSleep/afterWake + 编码器唤醒
- **媒体键**: 旋钮模式加入音量控制 (MEDIA_VOLUME_UP/DOWN), 已有库支持
- **外壳**: 3D打印或亚克力, 固定4屏+旋钮布局
- **PCB**: 原型验证后画正式 PCB 替代面包板
